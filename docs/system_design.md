# 系统设计文档 (System Design)

## 项目概述
能读取本地git未提交或者提交的具体commit的diff情况，在web页面进行展示，然后由人类进行review，最后将review结果以结构化的数据一次性导出，给到AI Coding工具进行问题修复。


## 技术栈
- 后端：go+gin框架。
- 前端：vue(vite) + tailwindcss 

### 提供设定
1. 后端通过`github.com/spf13/cobra`提供命令行入口
2. 后端通过go官方提供embed将前端页面打包到一个可执行二进制文件
3. 通过`wtf2pr web --port={port} --workdir={工作目录}`这样的指令启动服务

## 系统架构
1. 前端：负责展示后台提供的数据；负责用户在web页面添加review说明时，发送给后端，后端对当前review进行数据记录，
2. 后端：负责获取git diff或者git commit信息；接受前端review的信息；将整体信息格式化输出，支持markdown，json，xml等格式


## AI 开发守则
> 当 AI 依据本文档进行功能开发时，必须遵守以下规则：

1. 你要严谨认真的完成任务，在对于相关技术由疑问的时候，需要通过网络查询或者mcp工具查询获取更多信息
2. 前后端通信要以定义严格的结构体进行处理，不要出现临时定义临时使用的情况
3. 你要注意代码质量，该复用的代码要进行复用，保证一个功能只有一个函数实现的情况
4. 完成编码之后要进行代码review，自己对照目标找出问题并进行修复
